# Coding Patterns for Competitive Programming

Master essential patterns used in coding competitions and technical interviews.

## What are Coding Patterns?

Coding patterns are reusable problem-solving strategies that work for multiple types of problems. Learning these patterns helps you recognize problem structures and apply proven solutions quickly.

## Patterns Covered

### 01. Two Pointers
Use two pointers to traverse arrays/strings from different directions to solve problems efficiently.

**Best for:**
- Finding pairs in sorted arrays
- Container with most water
- Palindrome validation
- Merging sorted arrays

### 02. Sliding Window
Maintain a window of elements and slide it across the data structure to find optimal substrings/subarrays.

**Best for:**
- Maximum/minimum subarray problems
- Longest substring without repeating characters
- Permutation & anagram problems
- Fixed-size window calculations

[View Sliding Window ‚Üí](./02-sliding-window)

### 03. Binary Search
Efficiently search in sorted arrays by dividing search space in half.

**Best for:**
- Searching in sorted arrays
- Finding boundaries (first/last occurrence)
- Rotated array search
- Search space optimization

[View Binary Search ‚Üí](./03-binary-search)

### 04. DFS & BFS
Graph traversal patterns for exploring nodes in various orders.

**Best for:**
- Tree/graph traversal
- Connected components
- Shortest path (BFS)
- Topological sorting
- Backtracking problems

[View DFS & BFS ‚Üí](./04-dfs-bfs)

### 05. Dynamic Programming
Build solutions from smaller subproblems with memoization/tabulation.

**Best for:**
- Optimization problems
- Counting problems
- Value maximization
- Sequence-based problems

[View Dynamic Programming ‚Üí](./05-dynamic-programming)

1. Identify the data structure (array, string, tree, graph)
2. Determine the goal (find, maximize, count, etc.)
3. Recognize the pattern
4. Apply the template solution
