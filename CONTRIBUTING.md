# Contributing to Learn Coding

Thank you for your interest in contributing to Learn Coding! We welcome contributions from everyone.

## How to Contribute

### 1. Fork and Clone

```bash
# Fork the repository on GitHub, then:
git clone https://github.com/yourusername/learn-coding.git
cd learn-coding
```

### 2. Create a Feature Branch

```bash
git checkout -b feature/your-feature-name
```

### 3. Make Your Changes

- Keep code well-commented and readable
- Follow existing code style and conventions
- Test your examples thoroughly
- Update documentation if needed

### 4. Commit Your Changes

```bash
git add .
git commit -m "Add brief description of changes"
```

Write clear, descriptive commit messages:
- Use the imperative mood ("Add feature" not "Added feature")
- Be concise but descriptive
- Reference issues when applicable (#123)

### 5. Push and Create a Pull Request

```bash
git push origin feature/your-feature-name
```

Then create a Pull Request on GitHub with:
- Clear title describing the change
- Description of what was changed and why
- Reference to any related issues

## Code Guidelines

### Code Style
- Use consistent indentation (2 or 4 spaces)
- Keep lines reasonably short
- Use meaningful variable names
- Add comments for complex logic

### Documentation
- Comment your code, especially complex sections
- Add docstrings/JSDoc for functions
- Update README if adding new sections
- Include usage examples where helpful

### Testing
- Test your code before submitting
- Verify examples run correctly
- Check for common mistakes

## Reporting Issues

Found a bug? Have a suggestion? Here's how to report it:

1. **Check existing issues** first - your issue might already be reported
2. **Create a new issue** with:
   - Clear, descriptive title
   - Description of the problem
   - Steps to reproduce (if applicable)
   - Expected vs actual behavior
   - Any relevant code or screenshots

## Code of Conduct

Be respectful and constructive:
- Welcome diverse perspectives
- Be patient with newcomers
- Provide helpful feedback
- Keep discussions focused and professional

## Questions?

- Open a GitHub Discussion
- Create an issue with your question
- Check existing documentation
